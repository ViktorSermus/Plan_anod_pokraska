# Plan_anod_pokraska

Локальное приложение для:
- чтения данных из `ZNOM` (все файлы),
- чтения данных из `Reestr_ZNOM` (только самый свежий файл),
- агрегации и сохранения в SQLite,
- ручного ввода только поля `Вывезено` без потери при обновлениях.

## Запуск

### Вариант для пользователя (в один клик)

1. Убедиться, что установлен Python 3.11+ (с опцией "Add Python to PATH").
2. Запускать файл `start_app.vbs` (тихий запуск) или `start_app.bat`.
3. На первом запуске автоматически:
   - создается `.venv`,
   - устанавливаются зависимости,
   - открывается приложение в браузере.

### Ручной запуск (для разработчика)

1. Установить зависимости:
   - `pip install -r requirements.txt`
2. Проверить `config.json` (пути к папкам и БД).
3. Запустить UI:
   - `streamlit run streamlit_app.py`

## Как работает сохранение ручного поля

- Для строк рассчитывается `business_key` как хэш:
  - `№ заявки + Дата заявки + Наименование`.
- При обновлении master-данных:
  - данные пересчитываются из Excel,
  - `Вывезено` сохраняется из предыдущей версии по `business_key`,
  - исчезнувшие из источника строки помечаются `is_active = 0` (архив), а не удаляются.

## Примечания

- Исходные Excel-файлы не изменяются, чтение только read-only.
- Ошибочные/занятые файлы пропускаются и выводятся как предупреждения при обновлении.
